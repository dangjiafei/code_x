# 创作者中心 - 招募大厅与活动管理 PRD

## 一、需求背景

创作者中心面向医生创作者，用于承接平台发布的学术征集活动。平台后台创建活动后，厂家端可选择使用活动中的主题并配置目标医生条件；符合条件的医生会在创作者中心看到对应活动，并围绕开放主题进行创作投稿。

本期活动形态与历史活动不同：一个平台活动可能包含多个征稿主题，也可能覆盖医学播客、在线互动病例、手术/操作视频等多种内容形式。因此医生端不再以“活动类型”作为一级入口，而是统一以“平台活动”作为医生认知入口，再在活动详情中查看主题和投稿要求。

## 二、模块范围

一级系统：趣学术 - 创作者中心

一级菜单：

- 招募大厅
- 活动管理

涉及页面：

- 招募大厅列表页
- 活动管理页
- 活动详情页入口说明

不包含范围：

- 平台后台活动创建流程
- 厂家端活动使用配置流程
- 投稿编辑器、作品审核详情、采购结算等独立流程
- 移动端医生页面 PRD

## 三、核心业务规则

1. 医生端只体现平台活动，不体现厂家或企业信息。
2. 活动入口不再按医学播客、在线互动病例、手术/操作视频拆分 Tab。
3. 一个活动可以包含多个主题，医生可查看活动下全部主题。
4. 医生只可对符合招募条件且已开放的主题进行创作投稿。
5. 不可参与、未开放或已结束的主题可查看，但不提供投稿入口。
6. 招募大厅用于发现可参与或可关注的活动。
7. 活动管理用于管理医生已经参与过、投稿过或被纳入招募范围的活动。
8. 活动卡片统一展示活动封面，封面用于表达活动方向和专业感。
9. 卡片操作统一进入活动详情，主题级投稿、修改、查看审核结果等操作在详情页内处理。

## 五、整体信息架构

### 5.1 菜单结构

| 菜单 | 说明 |
|---|---|
| 招募大厅 | 医生发现平台招募活动的入口。 |
| 活动管理 | 医生查看自己参与中和历史活动的入口。 |

### 5.2 不再保留的入口

- 不再将“医学播客”“在线互动病例”等内容形式作为活动管理的一级 Tab。
- 不再展示“专家评审”菜单。
- 不再在医生端出现厂家、企业、采购、客户等表达。

### 5.3 医生端命名原则

- 使用“活动”“主题”“投稿”“创作”等医生容易理解的表达。
- 避免使用“采购”“企业选择”“目标医生条件”“平台匹配”等偏后台或厂家端视角的表达。
- 按钮优先表达医生当前可做的动作，如“查看活动”“查看详情”。

## 六、招募大厅

### 6.1 页面入口

菜单路径：创作者中心 - 招募大厅

### 6.2 页面目标

让医生快速看到当前平台正在组织的学术征集活动，了解本期重点活动、活动方向、可参与主题数量和剩余时间，并进入活动详情查看完整主题和投稿要求。

### 6.3 页面结构

| 区域 | 说明 |
|---|---|
| 本期重点 | 展示平台重点推荐的活动，突出活动名称、活动方向、内容形式、可参与主题、剩余时间和医学领域。 |
| 筛选区 | 支持搜索活动名称、按医学领域筛选。 |
| 快捷筛选 | 全部活动、即将截止、最新开放、我关注的。默认展示“我关注的”。 |
| 活动卡片列表 | 以 4 列卡片形式展示活动，卡片包含封面、状态、标题、简介、标签、关键指标和操作。 |
| 分页 | 数据较多时分页展示。 |

### 6.4 本期重点

本期重点展示当前最希望医生关注的活动。该区域用于提升活动的专业感和参与意愿，但不做过重的价值叙事。

字段说明：

| 字段 | 说明 |
|---|---|
| 标识 | 固定展示“本期重点”。 |
| 活动名称 | 展示重点活动标题。 |
| 活动简介 | 简短说明活动关注的临床场景或创作方向。 |
| 内容形式标签 | 展示该活动支持的内容形式，例如医学播客、在线互动病例、手术/操作视频。 |
| 可参与主题 | 展示医生当前可参与主题数 / 活动全部主题数。 |
| 剩余时间 | 展示距离投稿截止的剩余天数。 |
| 医学领域 | 展示活动所属医学领域。 |
| 查看活动 | 点击进入活动详情页。 |

交互规则：

- 点击“查看活动”进入活动详情。
- 本期重点不展示收藏按钮，避免主视觉区域操作过多。
- 本期重点数据来自平台活动配置和医生可参与主题计算结果。

### 6.5 搜索与筛选

#### 6.5.1 搜索框

| 项目 | 说明 |
|---|---|
| 占位文案 | 搜索活动名称 |
| 搜索范围 | 活动名称 |
| 触发方式 | 输入后实时筛选或点击搜索触发，具体由实现决定。 |
| 无结果 | 展示空状态，提示暂无匹配活动。 |

#### 6.5.2 医学领域筛选

筛选项：

- 全部医学领域
- 心血管
- 内分泌
- 儿科
- 骨科
- 神经科
- 呼吸科
- 肿瘤科

规则：

- 医学领域筛选与搜索条件叠加生效。
- 筛选后卡片列表只展示符合条件的活动。

#### 6.5.3 快捷筛选

| 筛选项 | 说明 |
|---|---|
| 全部活动 | 展示当前医生可查看的全部活动。 |
| 即将截止 | 展示临近投稿截止的活动。 |
| 最新开放 | 展示最近开放招募的活动。 |
| 我关注的 | 展示医生已收藏/关注的活动。默认选中。 |

规则：

- 默认进入页面时展示“我关注的”。
- 医生点击活动卡片上的收藏按钮后，当前列表需要立即刷新筛选结果。
- 若“我关注的”无数据，展示空状态并提供切换到“全部活动”的引导。

### 6.6 活动卡片

#### 6.6.1 卡片布局

- 默认桌面端一行 4 个卡片。
- 卡片宽度需保持统一，不因列表数量不足而拉伸占满整行。
- 屏幕缩小时可降为 3 列、2 列、1 列。
- 所有元素不使用阴影效果。
- 卡片整体使用白色背景，与活动管理保持一致。

#### 6.6.2 卡片字段

| 字段 | 说明 |
|---|---|
| 活动封面 | 展示平台配置或生成的活动封面。 |
| 活动状态 | 展示在封面右上角，例如招募中、即将截止、即将开放、已结束。 |
| 收藏按钮 | 展示在封面区域内，避免单独占一行。 |
| 活动名称 | 展示平台活动标题。 |
| 活动简介 | 简短说明活动适合的创作方向。 |
| 内容形式标签 | 展示活动包含的内容形式。 |
| 可参与主题 | 展示医生可参与主题数 / 全部主题数。 |
| 剩余时间或开始时间 | 进行中活动展示剩余时间，即将开放活动展示开始时间。 |
| 医学领域 | 展示活动所属领域，字号需小于标题。 |
| 查看活动 | 进入活动详情页。 |
| 去参与 | 当活动存在可投稿主题时展示，点击也进入活动详情，由详情页承接具体主题选择。 |

### 6.7 活动状态

| 状态 | 说明 | 卡片表现 |
|---|---|---|
| 招募中 | 当前活动在投稿期内，且医生存在可参与主题。 | 展示“查看活动”“去参与”。 |
| 即将截止 | 活动临近投稿截止。 | 状态标签突出提醒，仍可进入详情参与。 |
| 即将开放 | 活动已发布但投稿入口未开放。 | 仅展示“查看活动”，不可投稿。 |
| 已结束 | 活动投稿期已结束。 | 可查看详情，不可投稿。 |

### 6.8 收藏逻辑

| 操作 | 说明 |
|---|---|
| 收藏 | 点击空心收藏图标后，活动进入“我关注的”。 |
| 取消收藏 | 点击实心收藏图标后，从“我关注的”移除。 |
| 默认筛选 | 页面默认展示“我关注的”活动。 |

注意：

- 收藏仅影响医生个人视图，不影响活动招募状态。
- 收藏不是报名，不代表医生已参与该活动。

### 6.9 空状态

| 场景 | 文案建议 |
|---|---|
| 搜索无结果 | 暂无匹配的活动 |
| 我关注的无数据 | 暂无关注的活动，可切换到全部活动查看 |
| 全部活动无数据 | 暂无可查看的招募活动 |

## 七、活动管理

### 7.1 页面入口

菜单路径：创作者中心 - 活动管理

### 7.2 页面目标

帮助医生管理自己已参与或有投稿记录的活动。医生可查看活动当前状态、已投稿数量、医学领域和历史记录，并进入活动详情处理主题级操作。

### 7.3 页面结构

| 区域 | 说明 |
|---|---|
| Tab | 我的活动、已结束。 |
| 筛选区 | 搜索活动名称、按医学领域筛选。 |
| 活动卡片列表 | 展示医生相关活动卡片。 |

### 7.4 Tab 规则

| Tab | 说明 |
|---|---|
| 我的活动 | 展示待投稿、审核中、未通过、已完成等未归档活动。 |
| 已结束 | 展示活动已结束的历史活动。 |

规则：

- 不展示“可参与活动”Tab。
- “可发现的新活动”统一放在招募大厅，不放在活动管理。
- Tab 数量展示当前分类下的活动数量。

### 7.5 筛选与搜索

#### 7.5.1 搜索框

| 项目 | 说明 |
|---|---|
| 占位文案 | 搜索活动名称 |
| 搜索范围 | 活动名称 |
| 生效范围 | 当前 Tab 内活动卡片 |

#### 7.5.2 医学领域筛选

我的活动和已结束均使用医学领域筛选。

筛选项：

- 全部医学领域
- 心血管
- 内分泌
- 神经科
- 消化科
- 呼吸科
- 骨科
- 肿瘤科
- 皮肤科
- 儿科

不需要的筛选：

- 不需要投稿状态筛选。
- 不需要内容形式筛选。
- 不需要“适合我”筛选。

### 7.6 活动卡片布局

- 我的活动和已结束使用同一套卡片尺寸。
- 卡片不因某个 Tab 数据较少而拉伸。
- 默认桌面端一行 4 个卡片。
- 屏幕变窄时按响应式规则降列。
- 卡片不使用阴影。
- 卡片底部操作统一为“查看详情”。

### 7.7 我的活动卡片字段

| 字段 | 说明 |
|---|---|
| 活动封面 | 展示活动封面。 |
| 当前状态 | 展示在封面右上角。 |
| 活动名称 | 展示平台活动标题。 |
| 主题提示 | 展示医生当前关联的主题信息，例如“投稿主题：冠心病诊疗新进展”或“选择主题：疑难诊断路径复盘”。 |
| 内容形式标签 | 展示该活动包含的内容形式。 |
| 已投稿 | 展示医生在该活动下的投稿数量。 |
| 当前状态 | 待投稿、审核中、未通过、已完成。 |
| 医学领域 / 剩余时间 / 处理方式 | 根据活动状态展示最有用的信息。 |
| 查看详情 | 点击进入活动详情页。 |

### 7.8 已结束卡片字段

| 字段 | 说明 |
|---|---|
| 活动封面 | 展示活动封面。 |
| 当前状态 | 固定展示“已结束”。 |
| 活动名称 | 展示平台活动标题。 |
| 活动方向 | 简短展示该活动历史方向。 |
| 内容形式标签 | 展示活动包含的内容形式。 |
| 全部主题 | 展示活动主题总数。 |
| 我的投稿 | 展示医生在该活动下的投稿数量。 |
| 当前状态 | 已结束。 |
| 查看详情 | 点击进入活动详情页，可回看活动信息和历史投稿。 |

### 7.9 活动状态

| 状态 | 说明 | 医生可操作 |
|---|---|---|
| 待投稿 | 医生存在可参与主题，但还未提交投稿。 | 查看详情，在详情页选择主题投稿。 |
| 审核中 | 医生已提交投稿，内容正在审核或整理。 | 查看详情，查看投稿状态。 |
| 未通过 | 医生有投稿未通过。 | 查看详情，在详情页查看具体投稿和原因。 |
| 已完成 | 医生在该活动下的投稿已完成处理。 | 查看详情，查看活动和投稿记录。 |
| 已结束 | 活动投稿期已结束。 | 查看详情，回看活动信息和历史投稿。 |

颜色规则：

- 待投稿：中性或蓝色。
- 审核中：蓝色或紫色。
- 未通过：红色，需与其他状态明显区分。
- 已完成：绿色。
- 已结束：灰色。

### 7.10 操作规则

活动管理卡片只保留“查看详情”。

不在卡片上展示以下操作：

- 查看投稿
- 继续创作
- 查看修改意见
- 立即投稿

原因：

- 一个活动下可能有多个主题。
- 一个主题下可能有多篇投稿。
- 投稿、修改、审核结果需要在活动详情中结合具体主题展示，卡片层级无法表达清楚。

### 7.11 活动详情跳转规则

| 来源 | 点击操作 | 目标 |
|---|---|---|
| 招募大厅 | 查看活动 | 活动详情页 |
| 招募大厅 | 去参与 | 活动详情页，并可在详情页定位到可参与主题区域 |
| 活动管理 - 我的活动 | 查看详情 | 活动详情页 |
| 活动管理 - 已结束 | 查看详情 | 活动详情页，历史回看状态 |

活动详情页需要承担：

- 展示活动整体信息。
- 展示活动全部主题。
- 标记医生可创作主题、已投稿主题、未开放主题和已结束主题。
- 展示主题级投稿入口。
- 展示主题级投稿记录和审核状态。

## 九、权限与可见性

1. 医生只能看到自己可查看的活动。
2. 医生是否可投稿由主题开放状态和医生条件匹配结果决定。
3. 医生端不展示企业名称、厂家名称、采购数量、目标医生条件等信息。
4. 已结束活动仅可查看，不可新增投稿。
5. 未开放主题仅可查看，不可投稿。

## 十、异常与边界场景

| 场景 | 处理 |
|---|---|
| 活动已结束但医生从旧链接进入 | 展示活动详情，投稿入口置灰。 |
| 医生无可参与主题 | 可查看活动和主题，但不展示投稿入口。 |
| 活动被平台停用 | 招募大厅不展示；活动管理中如有历史记录，可展示为不可投稿状态。 |
| 收藏活动后活动结束 | 招募大厅按筛选规则处理；活动管理中如有参与记录进入已结束。 |
| 医生有多个主题投稿 | 卡片只展示活动级汇总，详情页展示主题级明细。 |

## 十二、待确认事项

1. “我关注的”是否需要跨端同步到移动端医生侧。
