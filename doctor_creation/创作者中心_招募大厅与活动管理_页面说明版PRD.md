# 创作者中心 - 招募大厅与活动管理 PRD（页面说明版）

## 一、文档说明

本文档基于创作者中心现有页面梳理，用于描述“招募大厅”和“活动管理”模块的页面结构、展示内容、字段、筛选项、操作、弹窗/提示和页面跳转。

本文档不包含技术实现、接口定义、数据表结构、埋点方案等技术内容。

说明：本文档只描述招募大厅、活动管理和活动详情页。活动详情页中的“进入工作台”仅描述为进入对应内容类型的创作/投稿工作台，不展开工作台页面内容。

## 二、模块定位

招募大厅用于医生/创作者浏览平台开放的学术征集活动，按活动查看可参与主题，并进入对应内容类型的创作/投稿流程。

活动管理用于医生/创作者查看自己已参与、待投稿、审核中、未通过、已完成或已结束的活动，并回看活动详情和历史投稿。

## 三、菜单结构

入口：创作者中心侧边栏

| 菜单 | 页面 | 说明 |
| --- | --- | --- |
| 招募大厅 | 招募大厅页 | 浏览可参与的活动，支持搜索、筛选、收藏、查看活动和去参与。 |
| 活动管理 | 活动管理页 | 管理我的活动和已结束活动，支持搜索、筛选和查看详情。 |

## 四、页面清单

| 页面 | 说明 |

## 五、招募大厅页

### 5.1 页面目标

招募大厅用于医生/创作者浏览平台活动，快速找到可参与、即将截止、最新开放或已关注的活动，并进入活动详情或直接参与。

### 5.2 页面组成

| 区域 | 说明 |
| --- | --- |
| 筛选区 | 包含活动名称搜索、医学领域筛选和快捷筛选。 |
| 活动卡片列表 | 以卡片形式展示活动。 |
| 分页区 | 展示活动总数和分页按钮。 |

说明：页面顶部不展示横向重点活动推荐区，进入页面后直接展示筛选区和活动卡片列表。

### 5.3 搜索与筛选

#### 搜索框

| 项目 | 说明 |
| --- | --- |
| 占位文案 | 搜索活动名称 |
| 搜索范围 | 活动名称 |
| 交互 | 输入关键词后，活动卡片按活动名称进行筛选。 |
| 结果反馈 | 有关键词时，分页信息显示“找到 X 个活动”。 |

#### 医学领域筛选

筛选项：

| 选项 |
| --- |
| 全部医学领域 |
| 心血管 |
| 内分泌 |
| 儿科 |
| 骨科 |

交互说明：选择医学领域后，活动列表按医学领域筛选。

#### 快捷筛选

快捷筛选按钮：

| 按钮 | 说明 |
| --- | --- |
| 全部活动 | 展示全部活动。 |
| 即将截止 | 展示截止时间较近的活动。 |
| 最新开放 | 展示最近开放的活动。 |
| 我关注的 | 展示已收藏活动，默认选中。 |

交互说明：

| 操作 | 说明 |
| --- | --- |
| 点击快捷筛选 | 当前筛选按钮高亮，活动列表按对应条件刷新。 |
| 选择“我关注的” | 仅展示收藏状态为已收藏的活动。 |

### 5.4 活动卡片

每张活动卡片展示：

| 字段 | 说明 |
| --- | --- |
| 活动封面 | 展示活动视觉封面。 |
| 活动状态 | 展示招募中、即将截止、即将开放等状态。 |
| 收藏按钮 | 展示星标，支持收藏/取消收藏。 |
| 活动名称 | 展示平台活动标题。 |
| 活动简介 | 简要说明活动征集方向。 |
| 内容类型标签 | 如医学播客、在线互动病例、手术/操作视频。 |
| 可参与主题 | 展示可参与主题数量，如 2/5。 |
| 剩余时间/开始时间 | 展示剩余时间或开放时间。 |
| 医学领域 | 展示活动所属领域。 |
| 操作按钮 | 查看活动、去参与。 |

收藏按钮：

| 状态 | 展示 | 点击后 |
| --- | --- | --- |
| 未收藏 | ☆ | 切换为已收藏。 |
| 已收藏 | ★ | 切换为未收藏。 |

操作按钮：

| 操作 | 展示条件 | 点击后 |
| --- | --- | --- |
| 查看活动 | 所有活动可展示 | 进入活动详情页。 |
| 去参与 | 已开放且可参与活动展示 | 进入活动详情页，并由用户在详情页选择主题进入对应工作台。 |

说明：即将开放活动可以只展示“查看活动”，不展示“去参与”。

### 5.5 分页

分页区展示：

| 内容 | 说明 |
| --- | --- |
| 活动总数 | 默认显示“共 X 个活动”。 |
| 搜索结果数 | 搜索时显示“找到 X 个活动”。 |
| 上一页 | 第一页时置灰。 |
| 页码 | 当前页高亮。 |
| 下一页 | 点击进入下一页。 |

## 六、活动管理页

### 6.1 页面目标

活动管理用于创作者查看自己参与过或正在参与的活动，并按“我的活动”和“已结束”进行管理。

### 6.2 Tab 结构

活动管理包含 2 个 Tab：

| Tab | 数量展示 | 说明 |
| --- | --- | --- |
| 我的活动 | 我的活动 8 | 展示当前可操作、待投稿、审核中、未通过、已完成等活动。 |
| 已结束 | 已结束 4 | 展示已结束活动，可回看活动信息和历史投稿。 |

交互说明：

| 操作 | 说明 |
| --- | --- |
| 点击 Tab | 切换活动列表内容和对应筛选区。 |
| 当前 Tab | Tab 高亮，并显示对应内容区。 |

### 6.3 我的活动筛选区

| 控件 | 说明 |

## 七、活动详情页

### 7.1 页面目标

活动详情页用于展示单个活动的完整信息，包括活动基础信息、活动概览、征集主题、我的投稿、投稿规范和个人参与概览。创作者在此页面选择具体主题，并进入对应内容类型的工作台。

说明：本文档不展开工作台页面。点击“进入工作台”后，按内容类型进入既有创作/投稿流程，例如医学播客工作台、在线互动病例工作台、手术/操作视频工作台。

### 7.2 顶部操作

### 7.8 投稿规范

以列表展示活动统一投稿规则：

| 规则 |
| --- |
| 作品需基于真实临床经验，病例资料必须完成患者隐私脱敏。 |
| 内容需体现明确诊疗思路，包含关键检查、诊断依据、治疗方案和复盘总结。 |
| 同一作品只能投稿至一个开放主题，避免同一内容重复提交。 |
| 投稿前需完成原创声明与身份确认，平台确认后进入平台学术内容库。 |
