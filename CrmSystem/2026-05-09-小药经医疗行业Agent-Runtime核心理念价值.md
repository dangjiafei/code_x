# 小药经医疗行业 Agent Runtime 核心理念价值

日期：2026-05-09

## 0. 文档目的

回答四个问题：

1. 小药经到底是什么；
2. 它为什么在医药行业需要；
3. 它给 MR、DM、药企和趣学术带来什么核心价值；
4. 它与普通 AI 工具、知识库、CRM 的本质区别是什么。

## 1. 一句话表达

> 小药经是医药学术推广智能助理，帮助医药代表和地区经理安全合规地处理医生观念管理、医学信息传递和学术动作规划等复杂任务。

内部产品解释：

> 小药经医疗行业 Agent Runtime，是支撑这套智能助理能力的底层 AI 运行逻辑，负责读取上下文、识别认知差距、规划合规学术动作，并沉淀可解释、可审计、可复盘的过程证据。

## 2. 核心判断

小药经真正要解决的问题是：

> 在强合规环境下，一线医药代表和地区经理如何把学术推广从经验、人情和费用驱动，转向基于医学证据、医生认知差距和过程留痕的医生观念管理。

这件事过去做不起来，原因不是企业不知道要合规，也不是代表不知道要讲学术，而是缺少一套能把“医学证据、医生理解、代表动作、合规证明”串起来的低成本操作系统。

小药经的价值，就在于把这套复杂链路变成 MR 和 DM 能够日常使用的智能助理能力。

## 3. 产品存在的行业前提

### 3.1 医药代表角色正在被重新定义

传统医药代表曾经承担销售推动、客户维护、资源协调等多重职能。在强合规反腐之后，代表的职业边界必须收窄并升级。

合理的新定义是：

> 医药代表应成为医生观念管理者，而不是处方利益支付者。

这意味着代表可以做：

- 传递医学信息；
- 解释指南、共识和循证证据；
- 识别医生认知差距；
- 围绕学术问题持续沟通；
- 回收医生反馈；
- 推动医生观念向更合理的医学认知靠近。

代表不能做：

- 将医生参与和处方、份额、销量直接挂钩；
- 用学术外衣包装回扣；
- 以费用驱动替代医学信息传递；
- 把医生观念管理变成隐形处方管理。

### 3.2 药企需要的不只是合规工具，而是合规学术推广体系

单点合规检查解决不了一线执行问题。

真正需要的是一套过程体系：

```text
医学证据
-> 标准认知
-> 医生当前观念
-> 认知 gap
-> 学术动作
-> 医生反馈
-> 证据链
-> 复盘
```

小药经 Agent Runtime 就是这套体系的一线操作层。

### 3.3 AI 的价值不只是提高效率，而是降低专业执行门槛

录音总结、资料搜索、RolePlay 都能提升效率，但效率不是小药经的最终价值。

真正的价值是：

> 让普通代表和地区经理也能低成本完成过去只有强医学能力、强管理能力和强运营支持下才能完成的学术推广动作。

## 4. 小药经 Agent Runtime 不是什么

为了避免产品失焦，必须先说明它不是什么。

### 4.1 不是普通 AI 问答

普通 AI 问答回答的是：

> 用户问什么，AI 答什么。

小药经 Agent Runtime 处理的是：

> 在某个医生、某个产品、某个医学证据和某个合规边界下，下一步应该发生什么学术动作。

### 4.2 不是传统知识库

传统知识库管理的是资料。

小药经管理的是：

- 医生怎么想；
- 医生应该理解什么；
- 中间差在哪里；
- 下一步该如何学术沟通；
- 沟通后如何证明。

### 4.3 不是 CRM

CRM 的底层心智是客户管理和过程管控。

小药经的底层心智是：

> 帮助代表完成更专业、更合规、更可证明的学术工作。

因此，它必须避免成为新的“查岗系统”。否则代表会自然抵触。

### 4.4 不是医生端医学问答

小药经 Agent Runtime 首先服务 MR、DM、医学运营和合规等内部角色。

它不直接面向医生开放式回答医学问题，也不自动生成未经审核的医学结论给医生。

### 4.5 不是费用支付或报销工具

小药经不能围绕“钱如何出去”建立产品心智。

它应该围绕：

> 医学目标为什么存在，学术动作为什么发生，过程如何被证明。

## 5. 核心产品理念

### 5.1 从“记录工作”到“运行工作”

小药经不是把会议和拜访记录下来就结束。

它要把记录转化为下一步动作：

```text
录音 / 笔记 / 会议
-> 医生观念
-> 认知 gap
-> 学术动作
-> 证据链
```

记录只是输入，运行才是价值。

### 5.2 从“资料搜索”到“标准认知判断”

知识库不应只是让代表查资料。

它更重要的作用是建立标准认知：

> 在某个疾病、产品或临床问题上，一个医生合理应该理解什么。

只有先有标准认知，系统才能判断医生当前观念是否存在差距。

### 5.3 从“AI生成内容”到“AI规划动作”

小药经的 AI 不应只是帮代表写总结、写话术、写材料。

更重要的是规划动作：

```text
这个医生为什么需要触达？
应该解决哪个认知 gap？
应该使用什么内容？
应该以什么方式沟通？
做完以后留下什么证据？
```

### 5.4 从“事后补证据”到“过程即证据”

合规不能依靠事后补材料。

小药经应该让证据在动作中自然产生：

- 为什么发起；
- 基于什么内容；
- 触达了谁；
- 医生如何反馈；
- 代表如何跟进；
- 是否止步于医生观念管理。

### 5.5 从“个人效率工具”到“组织能力基础设施”

小药经首先要让个人愿意用，但最终价值不止于个人效率。

它应该把个人的一线观察，逐步转化为组织可复盘的学术推广能力：

```text
代表记录
-> 医生观念卡
-> 区域观念地图
-> 内容 gap
-> 培训与陪访方向
-> 合规学术动作复盘
```

## 6. 核心对象

小药经 Agent Runtime 的核心不是“文档”，而是五个业务对象。

### 6.1 医生观念

医生观念是系统的中心对象。

它描述医生在某个疾病、产品或临床问题上的理解、疑问、倾向和认知状态。

没有医生观念，小药经只能做会议总结；有了医生观念，小药经才能持续管理学术推广。

### 6.2 标准认知

标准认知来自指南、共识、循证证据、已审核医学内容和企业医学策略。

它不是 AI 自己编出来的答案，而是可追溯、可审核、可解释的医学依据。

### 6.3 认知 gap

认知 gap 是医生当前观念和标准认知之间的差距。

它是学术动作发生的理由。

没有 gap，触达就容易变成泛泛拜访；有了 gap，学术动作才有必要性和可解释性。

### 6.4 学术动作

学术动作是围绕认知 gap 产生的合规行为。

它可以是一次拜访准备、一次指南更新传递、一次医生观念调研、一次 RolePlay 演练、一次趣学术拜访卡触达，或一次 DM 陪访。

### 6.5 证据链

证据链证明学术动作为什么发生、如何发生、结果如何。

它是小药经和趣学术之间的关键连接点。

## 7. 核心运行逻辑

小药经底层 Agent Runtime 的核心循环是：

```text
读取上下文
-> 建立标准认知
-> 判断医生观念
-> 识别认知 gap
-> 规划学术动作
-> 调用工具执行
-> 检查合规边界
-> 回写观念变化
```

这套逻辑的关键不是“AI回答得多好”，而是：

> AI 能否在正确的业务上下文中，围绕医生观念管理持续推动下一步动作。

## 8. 工作区只是形态，行业逻辑才是壁垒

小药经可以借鉴 Claude / Cursor 的工作区形态：

- 文件；
- 浏览器；
- 知识库；
- 对话；
- 工具调用；
- skill 沉淀；
- 多端同步。

但这些只是底层形态，不是最终壁垒。

真正的壁垒在于：

- 医生观念对象；
- 标准认知模型；
- 认知 gap 识别；
- 合规学术动作规划；
- 趣学术执行与证据链；
- 医疗行业作业 skill。

所以小药经不能停留在“医疗版 Claude”。

更准确的理解是：

```text
底层像 Claude
对象是医疗
动作接趣学术
价值是医生观念管理
```

## 9. 对不同角色的核心价值

### 9.1 对 MR

小药经帮助代表回答：

```text
这个医生现在怎么想？
他缺什么认知？
我下次该讲什么？
怎么讲才专业、合规、有信心？
做完怎么证明？
```

代表获得的不是一个管控系统，而是一个专业工作助手。

### 9.2 对 DM

小药经帮助地区经理回答：

```text
区域医生观念卡在哪里？
哪些医生需要重点教育？
哪些代表能力不足？
哪些内容反复被问到？
本周区域应该抓什么学术动作？
```

DM 管理的不再只是代表行为，而是区域医生观念的推进状态。

### 9.3 对医学 / 内容运营

小药经帮助医学和内容运营回答：

```text
一线医生最常见的认知 gap 是什么？
哪些内容缺失？
哪些材料真正被代表使用？
哪些问题需要补充医学支持？
```

内容运营不再只是上传资料，而是持续维护标准认知与内容供给。

### 9.4 对药企和合规

小药经帮助企业回答：

```text
学术动作为什么发生？
它基于什么医学依据？
是否止步于医生观念教育？
过程有没有证据？
结果能否复盘？
```

企业获得的是一套合规学术推广的过程解释能力。

## 10. 与趣学术的关系

小药经和趣学术不是两个并列工具，而是一套闭环中的不同层。

```text
小药经：判断、规划、准备、演练、回写
趣学术：执行、触达、互动、留痕、审计、复盘
```

小药经负责把一线碎片信息转化为合规学术动作。

趣学术负责让这些动作真实发生，并形成证据链。

二者合在一起，才能完成：

> 从医生观念 gap 到合规学术动作，再到证据链和复盘的完整闭环。

## 11. 核心边界

小药经 Agent Runtime 必须有明确边界。

它做：

- 医生观念评估；
- 标准认知整理；
- 认知 gap 判断；
- MR 拜访准备；
- RolePlay 演练；
- 学术动作建议；
- 趣学术任务衔接；
- 证据链回写；
- DM 区域诊断。

它不做：

- 医生端开放式 AI 医学问答；
- 自动生成未经审核的医学结论给医生；
- 处方建议；
- 销量预测驱动的医生触达；
- 费用支付包装；
- 回扣场景设计；
- 替代医学、合规、法务最终判断。

## 12. 核心评价标准

判断小药经 Agent Runtime 是否成立，不应只看：

- 录音次数；
- 总结次数；
- 知识库文件数；
- AI 问答次数。

更应看：

- 是否形成医生观念卡；
- 是否识别出可信的认知 gap；
- 是否生成可解释的学术动作；
- 是否让 MR 更愿意使用；
- 是否让 DM 看清区域学术重点；
- 是否能接入趣学术执行；
- 是否自然产生证据链；
- 是否证明学术推广止步于医生观念教育。

## 13. 最核心的产品信念

小药经的核心信念不是“AI 提高效率”，而是：

> AI 可以把医药代表和地区经理从经验型、碎片化、低证据的工作方式中解放出来，让他们具备低成本运行合规学术推广的能力。

再进一步说：

> 小药经的使命，不是让代表多用一个工具，而是让代表和 DM 有能力在新合规时代成为真正的医生观念管理者。

## 14. 最终表达

> 小药经是医药学术推广智能助理，帮助医药代表和地区经理安全合规地处理医生观念管理、医学信息传递和学术动作规划等复杂任务。它以医学证据和标准认知为依据，识别医生认知 gap，规划合规学术动作，并衔接趣学术完成执行、留痕和复盘。它的价值不是替代表达，也不是增加管控，而是让 MR 和 DM 低成本具备专业、合规、可证明的学术推广能力。
