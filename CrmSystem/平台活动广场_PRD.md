# 平台活动广场 PRD

## 一、需求背景

平台活动广场为本期平台管理后台新增模块，用于平台统一创建、管理和发布学术征集活动。平台创建的活动发布后，会进入厂家端“学术 SaaS - 活动广场”，厂家可选择使用活动并配置主题及目标医生条件；医生端只感知平台活动，根据招募条件看到可参与主题并进行创作投稿。

本模块需要支持平台活动从创建、发布、查看数据、停用管理，到接收厂家需求反馈的完整管理能力。

## 二、模块范围

一级菜单：平台活动广场

二级菜单：

- 活动列表
- 客户需求反馈

涉及页面：

- 活动列表页
- 新建活动页
- 活动详情页
- 客户需求反馈页

不包含范围：

- 厂家端活动使用配置的详细 PRD
- 医生端活动详情与投稿流程 PRD
- 内容审核、采购结算、传播执行等独立模块 PRD

## 三、核心业务规则

1. 平台活动由平台后台创建，厂家不可直接创建平台活动。
2. 一个平台活动可以包含多个征稿主题，主题可归属于不同内容形式，例如医学播客、在线互动病例、手术/操作视频。
3. 厂家选择使用活动时，按主题进行选择和配置，平台根据厂家配置招募符合条件的医生。
4. 医生端不体现厂家信息，医生看到的是平台发布的活动和可参与主题。
5. 厂家提交的客户需求反馈只作为平台需求输入，不直接触发新建活动或新增主题。
6. 客户需求反馈状态仅对厂家展示：待处理、已采纳、未采纳。

## 四、页面说明

| 序号 | 页面 | 原型图 | 说明 |
|---|---|---|---|
| 1.1 | 平台活动广场 - 菜单 | 原型图待补 | 新增一级菜单“平台活动广场”。该菜单下有两个二级菜单：“活动列表”“客户需求反馈”。进入活动列表、新建活动、活动详情时，左侧菜单均高亮“活动列表”；进入客户需求反馈页时，高亮“客户需求反馈”。 |

## 五、活动列表页

### 5.1 页面入口

菜单路径：业务中心 - 平台活动广场 - 活动列表

### 5.2 页面目标

平台运营人员查看和管理平台创建的学术征集活动，快速了解活动状态、使用企业数、投稿情况和采购完成情况。

### 5.3 页面结构

| 区域 | 说明 |
|---|---|
| 筛选区 | 搜索活动名称，按创建时间、使用企业数、投稿数排序，按活动状态、活动类型、医学领域筛选。 |
| 视图切换 | 支持卡片视图和表格视图。默认卡片视图。 |
| 新建活动 | 点击进入新建活动页。 |
| 活动列表 | 展示活动卡片或表格数据。 |
| 分页 | 当数据超过一页时分页展示。 |

### 5.4 筛选与排序

- 搜索框：
  - 支持搜索活动名称。
  - 输入后点击搜索或回车触发查询。
- 排序：
  - 创建时间：最新、最早。
  - 使用企业数：多到少、少到多。
  - 投稿数：多到少、少到多。
- 活动状态筛选：
  - 全部状态
  - 草稿
  - 未开始
  - 进行中
  - 已结束
  - 已停用
  - 已作废
- 活动类型筛选：
  - 全部类型
  - 医学播客
  - 在线互动病例
  - 手术/操作视频
- 医学领域筛选：
  - 所有领域
  - 心血管疾病
  - 神经内科
  - 骨科
  - 消化内科
  - 内分泌科
  - 呼吸内科

### 5.5 卡片视图字段

| 字段 | 说明 |
|---|---|
| 活动封面 | 展示平台上传或生成的活动封面。 |
| 活动状态 | 显示在封面右上角，不同状态使用不同颜色。 |
| 活动名称 | 展示活动标题，超过展示范围需省略。 |
| 医学领域 | 展示活动所属医学领域。 |
| 招募时间 | 展示活动招募开始时间和结束时间。 |
| 使用企业数 | 已选择使用该活动的企业数量。 |
| 总投稿数 | 医生围绕该活动主题提交的内容总数。 |
| 采购完成数 | 厂家已完成采购的投稿内容数量。 |

### 5.6 表格视图字段

| 字段 | 说明 |
|---|---|
| 活动名称 | 活动标题，可点击进入活动详情。 |
| 医学领域 | 活动所属医学领域。 |
| 活动状态 | 草稿、未开始、进行中、已结束、已停用、已作废。 |
| 招募时间 | 活动招募时间范围。 |
| 使用企业数 | 已使用企业数量。 |
| 投稿数 | 医生投稿数量。 |
| 采购完成数 | 厂家采购完成数量。 |
| 创建人 | 活动创建人员。 |
| 创建时间 | 活动创建时间。 |
| 操作 | 查看详情、编辑、停用等。 |

### 5.7 活动状态说明

| 状态 | 说明 | 可操作 |
|---|---|---|
| 草稿 | 已保存但未发布。 | 编辑、发布、删除。 |
| 未开始 | 已发布，招募开始时间未到。 | 查看详情、编辑、停用。 |
| 进行中 | 招募时间内，厂家可使用，医生可被招募投稿。 | 查看详情、停用。 |
| 已结束 | 招募结束时间已过。 | 查看详情。 |
| 已停用 | 平台主动停用，厂家端不再可使用。 | 查看详情、恢复。 |
| 已作废 | 活动不可继续使用，仅保留历史记录。 | 查看详情。 |

### 5.8 交互说明

- 点击“新建活动”进入新建活动页。
- 点击活动卡片进入活动详情页。
- 切换卡片/表格视图时，当前筛选条件不清空。
- 状态颜色不能只依赖文字区分，需使用不同色彩标签。
- 活动名称过长时省略显示，鼠标悬停可展示完整名称。

## 六、新建活动页

### 6.1 页面入口

活动列表页点击“新建活动”进入。

### 6.2 页面目标

平台运营人员创建平台级学术征集活动，并配置活动基础信息、征稿主题、投稿要求、医生端概览信息和活动海报。

### 6.3 顶部操作

| 操作 | 说明 |
|---|---|
| 返回列表 | 返回活动列表页。未保存时需提示确认。 |
| 保存草稿 | 保存当前填写内容，活动状态为草稿。 |
| 发布活动 | 校验必填项，通过后发布活动。 |

### 6.4 创建步骤

| 步骤 | 名称 | 说明 |
|---|---|---|
| 第一步 | 基础信息 | 填写医学领域、活动名称、活动背景、活动目的、封面和招募时间。 |
| 第二步 | 征稿主题与投稿要求 | 配置活动包含的主题和统一投稿要求。 |
| 第三步 | 活动概览生成 | 生成医生端活动详情中的活动概览信息。 |
| 第四步 | 活动海报 | 基于活动信息生成移动端活动海报。 |

### 6.5 第一步：基础信息

| 字段 | 是否必填 | 规则 |
|---|---|---|
| 医学领域 | 是 | 下拉选择，支持新增领域。 |
| 活动名称 | 是 | 最多 50 字，不可与已有活动重名。 |
| 活动背景 | 否 | 用于描述活动背景，可作为 AI 生成概览参考。 |
| 活动目的与目标 | 否 | 描述活动目的和目标，例如沉淀真实病例经验、促进学术交流等。 |
| 活动封面 | 是 | 支持 jpg/png，建议 16:9，大小不超过 2MB。 |
| 招募时间范围 | 是 | 开始时间需早于结束时间。 |

### 6.6 第二步：征稿主题与投稿要求

#### 6.6.1 征稿主题

主题按内容形式分组展示：

- 医学播客
- 手术/操作视频
- 在线互动病例

每个主题包含：

| 字段 | 是否必填 | 说明 |
|---|---|---|
| 主题名称 | 是 | 最多 30 字，用于区分创作方向。 |
| 主题说明 | 是 | 最多 200 字，说明医生围绕什么临床方向创作。 |
| 创作重点 | 否 | 用于提示医生重点表达内容，如诊断依据、治疗取舍、随访管理。 |

主题操作：

- 新增主题：每个内容形式下可新增主题。
- 删除主题：至少保留一个主题。
- 上移/下移：用于调整同类型下主题顺序。
- 折叠/展开：支持按内容形式折叠主题区域。
- 数量限制：单个内容形式建议最多 10 个主题。

#### 6.6.2 投稿要求

投稿要求为活动级统一要求，不按主题单独设置。

默认要求示例：

- 内容完整：说明病例背景、诊断依据、治疗过程、关键判断和复盘思考。
- 隐私保护：患者姓名、身份证号、联系方式、影像编号等隐私信息需脱敏。
- 学术价值：突出诊疗思路、决策依据、经验启发或特殊病例特点。

操作：

- 新增要求
- 删除要求
- 修改要求标签
- 修改要求内容

规则：

- 至少保留一条投稿要求。
- 投稿要求为空时不能发布活动。

### 6.7 第三步：活动概览生成

页面目标：生成医生端活动详情页“本期征集重点”模块所需内容。

字段：

| 字段 | 说明 |
|---|---|
| 本期征集重点标题 | 医生端概览模块标题。 |
| 概览说明 | 简要说明本期活动覆盖方向，以及适合医生如何创作。 |
| 覆盖方向标签 | 展示活动覆盖的主要主题方向，可新增和删除。 |

AI 能力：

- 点击“AI 生成概览”后，系统根据基础信息、主题名称、主题说明和创作重点生成概览内容。
- 用户可手动修改 AI 生成结果。
- AI 生成结果不自动发布，需要用户确认后进入下一步。

展示规则：

- 该步骤内容会用于医生端活动详情页。
- 不展示给厂家端作为配置项。

### 6.8 第四步：活动海报

页面目标：生成厂家端和移动端可展示的活动封面/海报视觉。

页面结构：

| 区域 | 说明 |
|---|---|
| 左侧信息源 | 展示海报将使用的活动名称、医学领域、招募时间、主题数量、部分主题。 |
| 中间生成设置 | 选择海报风格，并填写 AI 生成要求。 |
| 右侧海报预览 | 展示生成后的移动端长图海报。 |

生成设置：

- 海报风格由系统提供固定选项。
- 主题数量、按钮文案、信息结构由模板固定。
- 用户可通过输入框描述希望的画面风格。

海报内容建议：

- 活动标题
- 活动概括短句
- 医学领域标签
- 招募时间
- 部分征稿主题
- 查看全部主题/去报名按钮

操作：

- 重新生成
- 保存草稿
- 发布活动

### 6.9 发布校验

发布活动前需校验：

- 医学领域已选择。
- 活动名称已填写且不重复。
- 活动封面已上传或已生成。
- 招募时间范围有效。
- 至少存在一个征稿主题。
- 每个主题至少填写主题名称和主题说明。
- 至少存在一条投稿要求。
- 活动概览内容已生成或已填写。
- 活动海报已生成。

发布成功后：

- 活动进入活动列表。
- 活动状态根据招募时间判断为“未开始”或“进行中”。
- 厂家端活动广场可见。

## 八、客户需求反馈页

### 8.1 页面入口

菜单路径：业务中心 - 平台活动广场 - 客户需求反馈

### 8.2 页面目标

平台接收厂家端提交的活动建议或新活动需求，查看需求详情，并决定是否采纳。

### 8.3 反馈来源

厂家端可提交两类反馈：

| 类型 | 说明 |
|---|---|
| 活动主题建议 | 针对已有平台活动提交建议，例如希望增加主题、调整主题方向等。 |
| 新活动需求 | 不关联已有活动，希望平台创建某类新活动。 |

### 8.4 状态规则

平台后台和厂家端统一仅展示三种状态：

| 状态 | 说明 |
|---|---|
| 待处理 | 厂家刚提交，平台尚未处理。 |
| 已采纳 | 平台认为该需求可采纳。 |
| 未采纳 | 平台认为该需求暂不采纳。 |

说明：

- 不展示“评估中”“处理中”“已处理”等内部状态。
- 采纳后不直接在本页面创建活动或新增主题。
- 后续是否创建活动或新增主题，由平台运营在线下或活动创建流程中处理。

### 8.5 页面结构

| 区域 | 说明 |
|---|---|
| 页面标题 | 客户需求反馈。 |
| 状态筛选 | 全部反馈、新活动需求、活动主题建议、待处理、已采纳、未采纳。 |
| 搜索区 | 支持搜索反馈内容、企业或活动名称。 |
| 状态下拉 | 支持按待处理、已采纳、未采纳筛选。 |
| 列表 | 展示反馈记录。 |
| 分页 | 多页数据分页展示。 |

### 8.6 列表字段

| 字段 | 说明 |
|---|---|
| 反馈类型 | 活动主题建议、新活动需求。 |
| 反馈内容 | 展示标题和简要描述。 |
| 关联活动 | 活动主题建议展示关联活动；新活动需求展示“-”。 |
| 提交企业 | 提交反馈的企业名称。 |
| 提交时间 | 企业提交时间。 |
| 状态 | 待处理、已采纳、未采纳。 |
| 操作 | 查看详情、采纳、不采纳。 |

### 8.7 操作规则

| 当前状态 | 操作 | 说明 |
|---|---|---|
| 待处理 | 查看详情 | 打开反馈详情弹窗或详情页。 |
| 待处理 | 采纳 | 状态变为已采纳。 |
| 待处理 | 不采纳 | 状态变为未采纳。 |
| 已采纳 | 查看详情 | 查看反馈详情和处理结果。 |
| 未采纳 | 查看详情 | 查看反馈详情和不采纳结果。 |

### 8.8 反馈详情

点击“查看详情”展示完整反馈信息。

字段：

- 反馈类型
- 关联活动
- 提交企业
- 提交人
- 提交时间
- 需求描述
- 当前状态

如果平台点击“不采纳”，需填写不采纳原因。

如果平台点击“采纳”，可选填写平台备注。

### 8.9 厂家端展示规则

厂家端“我的反馈”展示：

- 反馈标题
- 反馈类型
- 关联活动
- 提交时间
- 状态
- 查看详情

厂家端状态：

- 待处理
- 已采纳
- 未采纳

厂家端不展示平台内部处理流程。

## 十、异常与校验

### 10.1 活动创建异常

- 活动名称为空：提示“请输入活动名称”。
- 活动名称重复：提示“活动名称不可与已有活动重复”。
- 未选择医学领域：提示“请选择医学领域”。
- 未上传封面：提示“请上传活动封面”。
- 未配置主题：提示“请至少配置一个征稿主题”。
- 主题名称为空：提示“请填写主题名称”。
- 主题说明为空：提示“请填写主题说明”。
- 招募结束时间早于开始时间：提示“结束时间需晚于开始时间”。

### 10.2 客户需求反馈异常

- 采纳失败：提示“操作失败，请稍后重试”。
- 不采纳未填写原因：提示“请填写不采纳原因”。
- 反馈详情加载失败：提示“反馈详情加载失败，请重试”。

## 十二、待确认事项

1. 活动是否需要审核流程，还是平台创建后可直接发布。
2. 已发布活动是否允许编辑主题；若允许，是否影响已被厂家使用的主题。
3. 活动停用后，已使用该活动的厂家任务是否继续执行。
4. 客户需求反馈采纳后，是否需要关联后续创建的活动或新增主题。
5. 活动详情中的传播效果和调研评估数据来源是否已明确。
