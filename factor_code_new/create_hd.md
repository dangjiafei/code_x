一、页面整体结构
页面从上到下分为两个区域：

Tab 切换栏（两个标签：活动广场、我使用的）
内容区域（根据选中的 Tab 显示不同内容）


二、Tab 切换栏

活动广场（默认选中，高亮显示）

我使用的

切换规则：

点击任意标签，该标签变为高亮（例如文字加粗、底部有蓝色横线），内容区域刷新为对应的内容。

重要：活动广场标签下的内容（帮助提示、搜索框、大卡片、小卡片网格）不会出现在“我使用的”标签下。

四、“活动广场”标签下的内容
4.1 帮助提示行
位于 Tab 栏下方，内容区域的最上方。
一行浅灰色文字，包含两个可点击的链接：

“📘 什么是活动广场？”

“📖 如何使用？”

样式：文字较小，左侧加图标。两个链接之间用竖线分隔。
点击“什么是活动广场？”：弹出一个轻提示弹窗，内容为“平台精选各类学术活动，即选即用，快速获取医生投稿。”
点击“如何使用？”：弹出弹窗，展示三步引导：

选择感兴趣的活动

设置活动条件（如招募人数、截止时间）

平台自动启动招募，在“学术场景菜单”中跟踪进度

弹窗右上角有一个“关闭”按钮，点击关闭。

4.2 搜索和筛选栏
位于帮助提示行下方，占一整行。从左到右：

搜索框：左侧有放大镜图标，内部占位文字显示“搜索活动名称、主题”。用户输入关键词后，下方活动列表实时过滤（只显示名称或主题包含关键词的活动）。

领域筛选下拉框：显示“全部领域 ▼”。点击下拉，选项包括：全部领域、心血管、糖尿病、儿科、神经科、呼吸、肿瘤、皮肤科、消化内科。选中某个领域后，只显示该领域的活动。

4.3 推荐活动大卡片
位于搜索栏下方。只显示 1 个大卡片。

大卡片的选取规则（无需前端实现，后端返回时已决定）：

优先显示后台标记为“推荐”的活动；

如果没有推荐活动，则自动选择“招募剩余时间最短”且“使用团队数最多”的活动。

大卡片样式：

背景为浅渐变（例如从淡蓝色到白色），有轻微阴影。

内边距较大，比小卡片更突出。

卡片内包含以下信息：

标题：活动名称（如“全国心血管诊疗新进展学术研讨会”）

热度信息：“89个团队正在使用”（使用团队数）

倒计时标签：“仅剩5天”（招募结束剩余天数，若剩余天数少于3天用红色文字）

招募时间范围：“招募时间：2025-12-01至2025-12-31”

两个按钮并排：左侧“查看详情”（白底蓝字），右侧“选择使用”（蓝底白字）

交互：

点击“查看详情”：跳转到该活动的详情页面。

点击“选择使用”：弹出配置面板（见下文第4.5节）。

4.4 全部活动区域（小卡片网格）
位于大卡片下方。上方有一个小标题“全部活动”，左对齐。

布局：小卡片采用网格布局，一行显示 3 个卡片（在 PC 浏览器上）。卡片之间间距均匀。

每个小卡片内容：

活动名称（最多两行，超出省略）

领域标签（如“心血管”）

关键指标标签：根据活动属性显示不同图标+文字：

如果活动剩余天数 ≤ 3 天：显示“⏰ 仅剩 X 天”（红色）

如果活动发布未满 3 天：显示“🆕 新”

如果使用团队数 > 50：显示“🔥 热门”

否则只显示团队数：“XX个团队”

按钮“选择使用”（蓝色，较小）

小卡片排序规则（后端返回，前端只需展示）：

默认按招募开始时间倒序（最新发布的活动在前）。

用户可以使用筛选栏改变排序？暂不做排序切换，保持简单。

交互：

点击卡片任意非按钮区域：跳转到活动详情页。

点击“选择使用”按钮：弹窗现在弹窗的内容不要变。

五、“我使用的”标签下的内容
当用户点击“我使用的”标签时，内容区域显示以下内容。

5.1 搜索和筛选栏
与广场类似，但字段不同：

搜索框：占位文字“搜索我的活动”。按活动名称实时过滤。

状态筛选下拉框：选项包括“全部状态”“进行中”“已结束”。默认“全部状态”。

5.2 已使用活动列表
以列表形式（而非网格）展示每个用户已选择使用的活动。每个活动占据一个完整的横向卡片（宽度100%）。

卡片样式：白色背景，细边框，圆角，内边距。卡片之间间距 16px。

卡片内容（从左到右排列）：

左侧区域：

活动名称（大号字）

活动状态标签（进行中 / 已结束），不同状态用不同颜色圆点+文字表示。

中间区域（展示与该用户相关的数据）：

投稿数量：“投稿：X 篇”

进度条：用百分比进度条表示活动完成情况（例如已投稿人数 / 目标招募人数）。进度条为浅灰色底，蓝色填充。

招募剩余时间（进行中）或“已结束”文字。

右侧区域：操作按钮组，根据活动状态显示不同按钮：

状态 = 进行中：

空状态：如果用户没有使用任何活动，则显示一个占位框：

文字：“还没有使用活动”

一个按钮：“去活动广场看看”，点击后自动切换到“活动广场”标签。
