# 厂家端活动广场模块 PRD（页面说明版）

## 一、需求背景

厂家端新增“活动广场”模块，用于厂家浏览平台发布的学术征集活动，并选择合适的活动主题进行使用。厂家可在活动详情中查看活动信息、配置使用计划、查看活动参与明细和作品管理；也可在活动广场中查看“我使用的”活动和内容，以及向平台提交活动建议或新活动需求。

本 PRD 基于当前前端页面整理，主要用于配合页面原型图给开发说明页面结构、字段展示、操作行为、弹窗内容和筛选规则。

## 二、模块范围

菜单位置：厂家端 - 学术 SaaS - 活动广场

本期包含：

- 活动广场首页
- 我使用的
  - 按活动查看
  - 按内容查看
- 我的反馈
- 活动详情页
  - 活动信息
  - 活动参与明细
  - 作品管理
- 选择使用弹窗
- 活动参与明细内容弹窗
- 提交活动需求/活动建议弹窗
- 反馈详情弹窗

本期不包含：

- 平台端创建活动流程
- 医生端投稿流程
- 数据报表详情页
- 作品详情弹窗
- 学术传递完整流程

## 四、活动广场首页

### 4.1 页面入口

- 菜单路径：学术 SaaS - 活动广场
- 默认进入“活动广场”Tab。

### 4.2 页面结构

页面顶部包含 3 个一级 Tab：

- 活动广场
- 我使用的
- 我的反馈

右上角有“提交活动需求”按钮。

活动广场 Tab 下包含：

- 页面说明 Banner
- 筛选区
- 领域筛选区
- 活动卡片列表

### 4.3 顶部 Tab

| Tab | 点击后展示 | 说明 |
| --- | --- | --- |
| 活动广场 | 活动广场列表 | 默认选中，用于浏览平台活动 |
| 我使用的 | 已使用活动工作台 | 点击后展示“按活动查看 / 按内容查看”二级切换 |
| 我的反馈 | 反馈列表 | 展示厂家提交过的活动建议和新活动需求 |

切换规则：

- 当前选中的 Tab 需要高亮。
- 切换到“我使用的”或“我的反馈”后，活动广场 Banner 和活动卡片列表隐藏。
- 切回“活动广场”后，重新展示 Banner、筛选区、领域筛选和活动卡片列表。

### 4.4 筛选区

| 控件 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| 搜索活动名称 | 输入框 | 空 | 支持按活动名称模糊搜索 |
| 活动状态 | 下拉框 | 全部 | 支持按活动状态筛选 |
| 搜索 | 按钮 | - | 点击后执行筛选 |

活动状态下拉选项：

- 全部
- 进行中
- 未开始
- 已结束

搜索规则：

- 搜索框支持输入活动名称关键字。
- 搜索结果与状态筛选、领域筛选同时生效。
- 如果没有匹配结果，列表为空。

### 4.5 领域筛选区

领域筛选为按钮组，单选。

选项包括：

- 全部领域
- 心血管
- 糖尿病
- 儿科
- 呼吸
- 神经科
- 肿瘤

交互规则：

- 默认选中“全部领域”。
- 点击某个领域后，该领域按钮高亮。
- 领域筛选与搜索、状态筛选同时生效。

### 4.6 活动卡片列表

每个活动以卡片展示。

卡片字段：

| 字段 | 说明 |
| --- | --- |
| 活动封面 | 展示活动图片 |
| 活动状态 | 展示在封面右上角，如进行中、未开始、已结束 |
| 活动名称 | 展示活动标题 |
| 医学领域 | 展示活动所属领域，如心血管、糖尿病等 |
| 主题数量 | 展示该活动包含几个征稿主题 |
| 团队使用数 | 展示已有多少团队使用 |
| 紧急标识 | 当活动临近结束时展示，如“仅剩5天” |
| 招募时间 | 展示活动招募开始和结束日期 |
| 操作 | 查看详情 |

状态展示规则：

- 进行中、未开始、已结束使用不同颜色标签展示。
- 状态不能只依赖文字区分，需要有颜色区分。

操作说明：

| 操作 | 点击事件 |
| --- | --- |
| 查看详情 | 跳转到活动详情页，URL 携带活动 ID |

## 七、我使用的 - 按内容查看

### 7.1 页面结构

包含：

- 内容统计区
- 筛选区
- 学术内容列表
- 空状态

### 7.2 内容统计区

| 指标 | 说明 |
| --- | --- |
| 征集内容 | 当前厂家可查看的内容总数 |
| 待审核 | 内容状态为待审核的数量 |
| 已采购 | 内容状态为已采购或已进入采购后流程的数量 |
| 待传播 | 已采购但尚未完成传播的数量 |
| 已传播 | 已完成传播的数量 |

### 7.3 筛选区

| 控件 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| 搜索作品名称或活动名称 | 输入框 | 空 | 支持按作品名称、活动名称搜索 |
| 全部类型 | 下拉框 | 全部类型 | 支持按内容类型筛选 |
| 全部内容状态 | 下拉框 | 全部内容状态 | 支持按作品状态筛选 |
| 重置 | 按钮 | - | 清空搜索和筛选条件 |

内容类型选项：

- 全部类型
- 医学播客
- 互动病例
- 手术视频

内容状态选项：

- 全部内容状态
- 待审核

- 已采购

筛选规则：

- 搜索支持作品名称和活动名称。
- 点击重置后，搜索框清空，下拉框恢复默认值。
- 列表上方展示“当前展示 X 条”。

### 7.4 学术内容列表字段

| 列名 | 说明 |
| --- | --- |
| 医生信息 | 展示医生姓名或“医生信息采购后可见”，以及职称、科室、医院 |
| 报名时间 | 医生报名该活动或任务的时间 |
| 作品名称 | 展示作品标题、作品 ID、内容类型标签 |
| 所属活动 / 主题 | 展示活动名称和投稿主题 |
| 投稿时间 | 医生提交作品的时间 |
| 作品状态 | 待审核、审核通过、已采购、已驳回 |
| 已传播人数 | 作品已传播触达的人数 |
| 操作 | 详情、更多 |

- 未采购内容：医生姓名展示为“医生信息采购后可见”。
- 已采购内容：展示医生姓名。
- 职称、科室、医院可展示在姓名下方。

作品名称展示规则：

- 第一行展示作品名称。
- 第二行展示作品 ID。
- 第三行展示内容类型标签，如医学播客、互动病例、手术视频。

### 7.5 操作列

| 操作 | 点击事件 |
| --- | --- |
| 详情 | 在“我使用的 - 按内容查看”中打开学术内容详情弹窗 |
| 更多 | 展开更多操作菜单 |

更多菜单内容：

- 编辑：置灰不可点击
- 内容合适：置灰不可点击
- 内容不合适：置灰不可点击
- 学术传递：可点击，点击后提示已进入学术传递
- 撤回：可点击，点击后提示已撤回该作品
- 观看链接：可点击，点击后提示已打开观看链接
- 观看记录：可点击，点击后提示已打开观看记录
- 分享作品：置灰不可点击

菜单交互：

- 点击“更多”后在按钮附近展示菜单。
- 点击页面其他位置关闭菜单。
- 不可用操作需要置灰并禁止点击。

### 7.6 空状态

当筛选后没有内容时，展示文案：

- 没有符合筛选条件的学术内容

## 九、提交活动需求/活动建议弹窗

### 9.1 打开入口

| 入口 | 默认弹窗类型 |
| --- | --- |
| 活动广场右上角“提交活动需求” | 希望平台创建新活动 |
| 活动详情页“反馈建议” | 针对已有活动 |

### 9.2 弹窗结构

弹窗包含：

- 弹窗标题
- 反馈类型切换
- 关联活动字段
- 需求描述
- 取消按钮
- 提交给平台按钮

### 9.3 反馈类型切换

类型一：针对已有活动

- 用于提交主题补充、主题方向建议、投稿要求建议等。
- 展示“关联活动”字段。
- 弹窗标题为“提交活动建议”。
- 需求描述占位文案：请描述当前活动哪里需要补充，或为什么希望新增这个主题。

类型二：希望平台创建新活动

- 用于提交新的学术征集方向。
- 隐藏“关联活动”字段。
- 弹窗标题为“提交活动需求”。
- 需求描述占位文案：请描述希望平台创建什么活动、面向什么医生、解决什么业务需求。

### 9.4 字段说明

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 反馈类型 | 卡片切换 | 是 | 针对已有活动 / 希望平台创建新活动 |
| 关联活动 | 下拉框 | 条件必填 | 仅针对已有活动时展示 |
| 需求描述 | 文本域 | 是 | 填写活动建议或新活动需求 |

关联活动下拉选项示例：

- 心血管疾病病例征稿活动
- 糖尿病规范化治疗病例征集大赛
- 儿科呼吸系统疾病病例征集
- 神经科疑难病例讨论沙龙
- 骨科疾病病例征集活动

### 9.5 按钮说明

| 按钮 | 点击事件 |
| --- | --- |
| 取消 | 关闭弹窗，不保存内容 |
| 提交给平台 | 新增一条反馈记录，状态为待处理，并跳转到“我的反馈” |

提交后反馈记录生成规则：

- 如果是新活动需求，标题格式为“希望平台创建：xxx”。
- 如果是活动建议，标题格式为“活动建议：xxx”。
- 反馈状态默认为“待处理”。
- 提交时间展示为“刚刚”。

## 十二、活动详情页 - 活动信息

### 12.1 页面结构

活动信息 Tab 分左右两栏：

左侧：

- 活动概览
- 征稿主题
- 内容要求

右侧：

- 活动预览

### 12.2 活动概览

展示 3 个信息块：

| 信息块 | 展示内容 |
| --- | --- |
| 本期征集重点 | 展示本期活动重点文案 |
| 概览说明 | 展示活动说明段落 |
| 覆盖方向 | 以标签形式展示覆盖方向 |

覆盖方向示例：

- 冠心病诊疗
- 心力衰竭管理
- 心律失常诊治
- 复杂冠脉介入

展示要求：

- 三个信息块 UI 样式保持统一。
- “本期征集重点”标题字号不能过小，需和其他信息块标题保持一致。

### 12.3 征稿主题

征稿主题按内容类型分组展示。

内容类型包括：

- 医学播客
- 在线互动病例
- 手术视频

每个分组展示：

| 字段 | 说明 |
| --- | --- |
| 内容类型名称 | 如医学播客 |
| 主题数量 | 展示该类型下主题数量 |
| 主题名称 | 主题一、主题二等 |

- 取消
- 确定使用

- 医学播客

交互规则：

- 支持多选主题。

- 点击内容类型标题可折叠/展开该分组。
- 搜索框支持按主题名称或类型搜索。

## 十七、页面状态与空状态

### 17.1 活动广场无结果

触发场景：

- 搜索或筛选后无活动。

展示：

- 活动列表为空。
- 建议展示空状态文案：暂无符合条件的活动。

### 17.2 我使用的无活动

触发场景：

- 厂家尚未选择使用任何活动。

展示：

- 建议展示空状态文案：暂无已使用活动，可前往活动广场选择使用。

### 17.3 内容列表无结果

触发场景：

- 按内容查看、作品管理或活动参与明细弹窗筛选后无结果。

展示文案：

- 没有符合筛选条件的学术内容

### 17.4 选择使用弹窗未选主题

触发场景：

- 未选择任何主题点击确定使用。

提示：

- 请至少选择一个活动主题

### 17.5 目标采购数错误

触发场景：

- 某个主题目标采购数为空、小于 1 或大于 500。

提示：

- 请为「主题名称」输入有效的采购数量（1-500份）
