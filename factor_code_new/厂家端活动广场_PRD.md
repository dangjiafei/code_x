# 厂家端活动广场 PRD

## 一、需求背景

“活动广场”为厂家端本期新增模块，位于厂家端“学术 SaaS”菜单下。平台后台发布平台活动后，厂家可在活动广场中浏览活动、查看活动详情、选择使用活动主题，并配置采购数和目标医生条件。厂家提交使用后，平台根据配置生成招募任务，符合条件的医生可在创作者中心看到活动并投稿。

该模块还需要支持厂家查看已使用活动、查看征集内容、提交活动需求反馈，并查看反馈状态。

## 二、模块范围

一级菜单：学术 SaaS

二级菜单：活动广场

模块内 Tab：

- 活动广场
- 我使用的
- 我的反馈

涉及页面/弹窗：

- 活动广场列表页
- 活动详情页
- 选择使用 / 配置企业使用计划弹窗
- 我使用的 - 按活动查看
- 我使用的 - 按内容查看
- 我的反馈
- 提交活动需求 / 活动建议弹窗
- 反馈详情弹窗

不包含范围：

- 平台后台创建活动流程
- 医生端活动详情与投稿流程
- 财务结算、合同、发票等流程
- 内容审核规则的后台管理

## 三、核心业务规则

1. 厂家只能选择使用平台已发布的活动，不能直接创建平台活动。
2. 一个活动包含多个主题，厂家使用活动时必须选择具体主题。
3. 厂家可一次选择多个主题，每个主题需单独配置采购数和目标医生条件。
4. 已使用过的主题再次配置时需置灰或提示，避免重复配置。
5. 厂家提交使用后，平台按主题生成招募任务。
6. 医生投稿会被绑定到明确主题任务，避免一个投稿对应多个主题。
7. 厂家端可提交两类反馈：针对已有活动的活动建议、不关联活动的新活动需求。
8. 厂家端反馈状态仅展示：待处理、已采纳、未采纳。

## 五、活动广场 Tab

### 5.1 页面入口

菜单路径：学术 SaaS - 活动广场 - 活动广场 Tab

### 5.2 页面目标

厂家浏览平台发布的学术活动，快速判断活动是否适合使用，并进入详情完成主题选择和使用配置。

### 5.3 页面结构

| 区域 | 说明 |
|---|---|
| 顶部 Tab | 活动广场、我使用的、我的反馈。 |
| 提交活动需求按钮 | 位于页面右上角，用于提交不关联具体活动的新活动需求。 |
| Banner 区 | 展示活动广场说明和视觉引导，仅活动广场 Tab 显示。 |
| 筛选区 | 支持搜索活动名称/主题，按活动状态筛选。 |
| 医学领域筛选 | 独立一行展示领域快捷筛选。 |
| 活动卡片列表 | 展示活动封面、状态、名称、领域、主题数、使用团队数、招募时间和查看详情按钮。 |

### 5.4 筛选规则

搜索框：

- 支持搜索活动名称。
- 可扩展支持搜索主题名称。

活动状态：

- 全部
- 进行中
- 即将开始
- 已结束

医学领域：

- 全部领域
- 心血管
- 糖尿病
- 儿科
- 呼吸
- 神经科
- 肿瘤

### 5.5 活动卡片字段

| 字段 | 说明 |
|---|---|
| 活动封面 | 平台后台配置或生成的活动封面。 |
| 活动状态 | 进行中、即将开始、已结束。 |
| 活动名称 | 平台活动名称。 |
| 医学领域 | 活动所属领域。 |
| 主题数 | 活动包含的征稿主题数量。 |
| 使用团队数 | 已有多少企业/团队使用该活动。 |
| 招募时间 | 活动招募开始和结束时间。 |
| 查看详情 | 点击进入活动详情页。 |

### 5.6 交互说明

- 点击“查看详情”进入活动详情页。
- 点击活动卡片也可进入详情页。
- 活动卡片不展示“反馈建议”按钮，避免列表操作复杂。
- 已结束活动可查看详情，但是否允许选择使用需根据业务规则控制。
- 切换到“我使用的”或“我的反馈”时隐藏活动广场 Banner 和活动卡片列表。

## 七、配置企业使用计划弹窗

### 7.1 触发入口

活动详情页点击“选择使用”。

### 7.2 页面目标

厂家选择活动中的具体主题，并为每个主题配置采购数和目标医生条件。

### 7.3 弹窗结构

| 区域 | 说明 |
|---|---|
| 顶部说明 | 标题“配置企业使用计划”，说明先选主题，再配置采购数和招募条件。 |
| 产品标签 | 选择企业产品标签。 |
| 左侧选择主题 | 按内容形式展示主题，支持多选。 |
| 右侧配置已选主题 | 选中主题后展示该主题配置项。 |
| 批量应用 | 可将采购数和目标医生条件批量应用到已选主题。 |
| 底部按钮 | 取消、确定使用。 |

### 7.4 产品标签

| 字段 | 是否必填 | 说明 |
|---|---|---|
| 产品标签 | 是 | 从企业已有产品标签中选择，例如医疗设备、药品、耗材、服务。 |

### 7.5 主题选择

主题按内容形式分组展示：

- 医学播客
- 在线互动病例
- 手术视频

每个主题展示：

- 主题名称
- 主题说明
- 内容形式

规则：

- 支持多选主题。
- 已使用过的主题置灰或不可重复选择。
- 顶部展示已选主题数量。
- 支持搜索主题名称或类型。

### 7.6 已选主题配置

每个已选主题需配置：

| 字段 | 是否必填 | 说明 |
|---|---|---|
| 采购数 | 是 | 每个主题独立配置采购数量。 |
| 省份/城市 | 否 | 选择目标医生所在省份或城市。 |
| 医院等级 | 否 | 选择目标医生所在医院等级。 |
| 身份/职称 | 否 | 选择目标医生职称或身份。 |
| 科室 | 否 | 选择目标医生所属科室。 |

### 7.7 批量应用

批量应用区支持填写：

- 采购数
- 省份/城市
- 医院等级
- 身份/职称
- 科室

点击“应用到已选主题”后，将配置同步到所有已选主题。

规则：

- 批量应用只覆盖用户已填写的字段。
- 已单独修改过的主题配置也可被批量覆盖。

### 7.8 提交校验

点击“确定使用”前校验：

- 必须选择产品标签。
- 至少选择一个主题。
- 每个选中主题必须填写采购数。
- 采购数需为正整数。

提交成功后：

- 保存企业使用计划。
- 活动进入“我使用的”。
- 平台按主题生成招募任务。
- 医生投稿会绑定到对应主题任务。

## 九、我的反馈 Tab

### 9.1 页面入口

学术 SaaS - 活动广场 - 我的反馈 Tab

### 9.2 页面目标

厂家查看自己提交过的活动建议和新活动需求，以及平台处理状态。

### 9.3 列表字段

每条反馈为一条记录，列表只展示摘要信息。

| 字段 | 说明 |
|---|---|
| 反馈标题 | 系统根据需求描述生成摘要标题，或展示用户填写摘要。 |
| 反馈类型 | 活动建议、新活动需求。 |
| 关联活动 | 活动建议展示关联活动；新活动需求展示无。 |
| 提交时间 | 用户提交反馈的时间。 |
| 状态 | 待处理、已采纳、未采纳。 |
| 查看详情 | 打开反馈详情弹窗。 |

### 9.4 状态说明

| 状态 | 说明 |
|---|---|
| 待处理 | 平台尚未处理该反馈。 |
| 已采纳 | 平台已采纳该反馈。 |
| 未采纳 | 平台暂不采纳该反馈。 |

说明：

- 不展示平台内部处理过程。
- 不展示“评估中”“处理中”等状态。
- 列表中不展示过长需求描述。

### 9.5 查看详情

点击“查看详情”打开弹窗。

弹窗展示：

- 反馈标题
- 反馈类型
- 关联活动
- 提交时间
- 当前状态
- 需求描述全文

规则：

- 列表仅展示摘要，长文本必须在详情弹窗中展示。
- 详情弹窗只读，不支持编辑。

## 十、提交活动需求 / 活动建议弹窗

### 10.1 触发入口

| 入口 | 说明 |
|---|---|
| 活动广场右上角“提交活动需求” | 默认提交新活动需求，不关联具体活动。 |
| 活动详情页“反馈建议” | 默认提交针对当前活动的活动建议。 |
| 我的反馈中提交入口 | 可扩展支持提交新的活动需求。 |

### 10.2 弹窗结构

弹窗顶部支持选择反馈类型：

- 针对已有活动
- 希望平台创建新活动

### 10.3 针对已有活动

字段：

| 字段 | 是否必填 | 说明 |
|---|---|---|
| 关联活动 | 是 | 下拉选择平台活动；从活动详情进入时默认带入当前活动。 |
| 需求描述 | 是 | 描述希望补充或调整的内容，例如希望新增主题、补充方向等。 |

说明：

- 关联活动必须为选择项，不允许手动输入。
- 不单独填写反馈类型、医学领域、医生重点创作内容。

### 10.4 希望平台创建新活动

字段：

| 字段 | 是否必填 | 说明 |
|---|---|---|
| 需求描述 | 是 | 描述希望平台创建什么样的活动，以及业务背景。 |

说明：

- 新活动需求不关联活动。
- 不单独填写医学领域，平台可从需求描述中判断。

### 10.5 提交规则

- 需求描述不能为空。
- 针对已有活动时必须选择关联活动。
- 提交成功后，新增一条“待处理”反馈记录。
- 提交后自动切换到“我的反馈”Tab。

## 十一、权限与角色

| 角色 | 权限 |
|---|---|
| 企业管理员 | 可查看活动、选择使用活动、配置使用计划、查看内容、提交反馈。 |
| 企业运营人员 | 可查看活动、配置使用计划、处理内容和提交反馈。 |
| 只读角色 | 仅可查看活动广场、活动详情和我的反馈，不可提交使用或反馈。 |

## 十二、异常与校验

### 12.1 活动广场

- 活动列表加载失败：提示“活动加载失败，请刷新重试”。
- 搜索无结果：展示空状态“暂无符合条件的活动”。
- 活动已结束且不允许使用：隐藏或禁用“选择使用”。

### 12.2 配置企业使用计划

- 未选择产品标签：提示“请选择产品标签”。
- 未选择主题：提示“请至少选择一个活动主题”。
- 采购数为空：提示“请填写采购数”。
- 采购数不是正整数：提示“采购数需为正整数”。
- 已使用主题重复选择：提示“该主题已配置使用计划”。

### 12.3 我的反馈

- 需求描述为空：提示“请填写需求描述”。
- 反馈提交失败：提示“提交失败，请稍后重试”。

## 十四、待确认事项

1. 已结束活动是否允许厂家继续选择使用。
2. 已使用主题是否允许重复配置或追加采购数。
3. 产品标签来源是否固定为企业产品管理模块。
4. 使用计划提交后是否需要平台审核。

6. 我的反馈中是否需要展示平台回复内容，当前版本仅展示状态和详情。
